gundi

The rodent that unearths your oldest TODOs. Scans TODO, FIXME, HACK, XXX — then runs git blame so you know who left it and how long it's been rotting.

~/my-project
$ gundi .
Scanned 127 files · Found 4 items · Blamed in 142ms
TYPE FILE LINE AUTHOR AGE TEXT
──────────────────────────────────────────────────────────────────────────────────
TODO src/config.rs 201 alice 310d ago support YAML format
TODO src/parser.rs 42 alice 142d ago refactor this loop
FIXME src/handler.rs 87 rivka 63d ago handle timeout case
HACK tests/integration.rs 15 charlie 8d ago temporary workaround
Summary: 2 rotten · 1 aging · 1 fresh
Oldest: 310 days (alice · src/config.rs:201)

● Rotten · older than 90d ● Aging · 30-90d ● Fresh · under 30d

Install in one line

Requires Rust. No config, no YAML, no setup flags.

cargo install gundi
$ git clone https://github.com/iamkorun/gundi.git
$ cd gundi
$ cargo install --path .
Gate your CI → Star on GitHub

Four features. No fluff.

Scan fast, blame accurately, filter precisely, fail CI loudly.

01 · scan

Digs through every file in parallel

Walks your tree with rayon, honors .gitignore, and sniffs out the six classic debt tags: TODO, FIXME, HACK, XXX, BUG, NOTE.

~127 files in well under a second on a medium repo.

scan output
$ gundi --types todo,fixme
TODO src/lib.rs:12 refactor state machine
TODO src/db.rs:88 add retry logic
FIXME src/api.rs:203 race on shutdown
02 · blame

Runs git blame on every hit

Each TODO comes back with the author's git identity, the commit SHA, and how long it has been parked. The comment you wrote during your first sprint? Yeah. Still there. 310 days old.

Blame is cached per commit — re-scanning the same tree is near-instant.

blame output
author: alice <a@team.dev>
sha: 7b39c2a
date: 2025-06-08
age: 310 days
text: "support YAML format"
03 · filter

Slice by type, age, or author

Just want the rotten stuff? --min-age 90. Just FIXMEs? --types fixme. Just alice's debt? --author alice. Chain them. Pipe to jq.

Output as table, json, or markdown.

filtered
$ gundi --min-age 90 --format json
[
{
"type": "TODO",
"file": "src/config.rs",
"age_days": 310
}
]
04 · fail

Block merges on rot

Drop --fail-on 90 into your workflow and gundi exits non-zero the moment a TODO crosses your threshold. No config file. No drama. Just a hard stop.

See the CI recipe below ↓

ci run
$ gundi --fail-on 90
error: 2 items exceed threshold (90d)
· src/config.rs:201 (310d, alice)
· src/parser.rs:42 (142d, alice)
exit code: 1

Drop-in GitHub Actions recipe

Any TODO older than 90 days fails the check. Adjust the threshold to your tolerance.

.github/workflows/debt.yml 
name: Code Debt Check
on: [pull_request]

jobs:
  debt:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
        with:
          fetch-depth: 0  # git blame needs full history
      - run: cargo install gundi
      - run: gundi --fail-on 90 .
